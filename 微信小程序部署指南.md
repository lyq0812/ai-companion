# 微信小程序部署解决Timeout问题指南

## 问题描述
微信开发者工具显示 `Error: timeout`，主要是因为 web-view 访问外部地址的问题。

## 立即解决方案

### 方案1：检查微信开发者工具设置 (最简单)

1. 打开微信开发者工具
2. 点击右上角「详情」
3. 勾选「不校验合法域名、web-view（业务域名）、TLS 版本以及 HTTPS 证书」
4. 重新编译小程序

### 方案2：使用国内CDN部署

#### 部署到 Vercel (已完成)
- 地址：https://ai-companion-hazel-one.vercel.app/
- 状态：✅ 已部署

#### 备选部署方案

**Netlify (推荐国内访问速度)**
1. 访问 https://netlify.com
2. 用 GitHub 账号登录
3. 导入你的仓库
4. 配置：
   - Build command: `cd frontend && npm run build`
   - Publish directory: `frontend/dist`
5. 部署完成后获取地址

**Gitee Pages (国内访问最快)**
1. 将代码推送到 Gitee
2. 仓库设置 -> 服务 -> Gitee Pages
3. 部署目录选择 `frontend/dist`
4. 启动服务

### 方案3：本地开发调试

如果只是本地测试，可以用以下方法：

1. 在 `frontend` 目录运行：
```bash
npm run dev
```

2. 用内网穿透工具（如 ngrok）将本地端口映射到公网：
```bash
ngrok http 3000
```

3. 将获取的地址更新到 `pages/index/index.js` 的 `webViewUrl`

## 微信小程序web-view配置要求

### 正式发布要求
1. 业务域名需要在微信公众平台配置
2. 域名必须是 HTTPS
3. 需要下载校验文件放到服务器根目录

### 开发阶段
- 可以关闭域名校验（方案1）
- 使用已部署的 HTTPS 地址

## 测试步骤

1. 确保微信开发者工具「不校验合法域名」已勾选
2. 重新编译小程序
3. 查看 Console 标签，应该能看到 `=== 页面加载 ===` 等日志
4. 如果还出错，点击「重新加载」按钮

## 常见问题

- 考虑部署到国内服务器

**Q: 正式发布需要什么？**
